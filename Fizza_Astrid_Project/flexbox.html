Historia

Enligt wikipedia, fanns det intensiva användning av webbsidor i mobil agenter på 2000-talet. Den motiverad användning av flexibel och anpassbar "layouts" och samtidigt responsiva webbsidor på grund av olika skärmsstorlek. "Under 2010-talet inspirerade intensiv användning av populära JavaScript-layoutramar, som Bootstrap, CSS flexbox och specifikationer för rutnätlayout."(Wikipedia)

Begrepp

"De flesta webbsidor skrivs i en kombination av HTML (Hypertext Markup Language) och CSS (Cascading Style Sheets). I korthet specificerar HTML innehåll och logisk struktur på sidan, medan CSS anger hur det ser ut: dess färger, teckensnitt, formatering, layout och styling. CSS flexbox-layout är ett särskilt sätt att specificera layouten för HTML-sidor. En av de mest framträdande funktionerna i flexlayouten är dess förmåga att formpassas, baserat på visningsmiljön. Flexboxar kan justeras i storlek - antingen minska, för att undvika onödigt monopolisering av utrymme eller öka för att ge utrymme för att innehåll ska begränsas inom dess gränser. Dessutom är flexlayouten mindre begränsande med avseende på innehållsflöde än de, till exempel, för block- och inline-visningstyper, som i allmänhet är i en riktning. Faktum är att inte bara flexningsriktningsflödet kan specificeras, på stilnivå, som höger, vänster, uppåt eller nedåt; enskilda föremål i en flexcontainer kan också ordnas automatiskt och omordnas för att passa det tillgängliga layoututrymmet"(wikipedia)

CSS Flexbox Layout Module

"Innan Flexbox Layout-modulen fanns fyra layoutlägen:

  • Block, för sektioner på en webbsida
  • Inline, för text
  • Table, för tvådimensionell tabelldata
  • Positioned, för uttrycklig position för ett element

Den flexibla rutan layoutmodul gör det lättare att utforma flexibel responsiv layoutstruktur utan med hjälp av flottör eller positionering. " (w3schools)

Flex Container Och Flex Items

Flex container

Förälderelement som innehåller alla flexföremål. Använd CSS-visningsegenskapen, behållaren kan definieras som antingen flex eller inline-flex.

Flex item

Alla direkta barnelement som hålls i flexbehållaren betraktas som en flexartikel. All text inom behållarelementet är lindat i en okänd flexartikel.

Axlar

"Varje flexbox innehåller två axlar: huvud- och tväraxlar. Huvudaxeln är den axel som artiklarna står i linje med varandra. Tväraxeln är vinkelrätt mot huvudaxeln

Directions"

"Huvudstart- / huvudändssidorna bestämmer var de ska börja placera flexföremål i flexbehållaren, börjar från huvudstartänden och går till huvudänden. Cross-start / cross-end-sidorna bestämmer var flexlinjer fylls med flex-objekt från cross-start till cross-end"
